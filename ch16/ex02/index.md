# Graceful Shutdown

## 概要
処理中のリクエストがある状態でWebサーバーのプロセスが停止したときにエラーが発生するのを避けるため、段階を踏んでシャットダウンすることでエラーの発生を抑制する手法。

## Kubernetes / Amazon ECS でのシグナル

Kubernetes や Amazon ECS などの Docker ランタイム上でコンテナの Graceful Shutdown のために送信されるシグナル:

### 1. SIGTERM
- プロセスに対して「正常終了してください」という終了を要求するシグナル
- このシグナルを受け取ってから終了処理を開始できる
- キャッシュの書き出し、接続のクローズ、ログ保存などが可能
- ハンドリングをプログラム側で自由に定義できる

### 2. SIGKILL
- プロセスを強制終了させるシグナル
- SIGTERM送信後、一定時間(猶予期間)経っても終了しない場合に送信される
- プログラム側でキャッチ・無視できない
- データ保存や後処理ができない
- 猶予期間:
  - Kubernetes: デフォルト30秒
  - Amazon ECS: デフォルト30秒
  - Docker: デフォルト10秒

## Graceful Shutdown のフロー
1. SIGTERMを送る: コンテナ停止命令時に最初に送信
2. 一定時間待つ: プロセスが正常終了するまで猶予期間を設ける
3. 終わらなければSIGKILLを送る: 猶予期間内に終了しない場合は強制終了
