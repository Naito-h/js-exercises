## 単語の調査
### 標準入力(stdin)
プログラムが外部からデータを受け取るための入り口
- 主な標準入力はキーボード
- 例：ユーザーがターミナルに入力したテキスト

### 標準出力(stdout)
プログラムが通常の結果を出力するための出力先
- 主な標準出力はターミナル
- 例：`console.log()`

### 標準エラー出力(stderr)
プログラムがエラーメッセージを出力するための出力先
- 標準出力とは別のため、正常な出力とエラーを分離
- 主な標準エラー出力はターミナル
- 例：`console.error()`, エラーメッセージ

### リダイレクト
コマンドの入出力先を別の入出力先に変更することができる仕組み

| 記号 | 説明 | 例 |
|------|------|-----|
| `>` | 標準出力をファイルへ上書き | `command > output.txt` |
| `>>` | 標準出力をファイルへ追記 | `command >> output.txt` |
| `2>` | 標準エラー出力をファイルへ上書き | `command 2> error.txt` |
| `2>>` | 標準エラー出力をファイルへ追記 | `command 2>> error.txt` |
| `<` | ファイルを標準入力として渡す | `command < input.txt` |
| `2>&1` | 標準エラー出力を標準出力に合流 | `command > all.txt 2>&1` |
| `&>` | 標準出力とエラー出力を両方リダイレクト | `command &> all.txt` |

### パイプ
コマンドの出力を別のコマンドの入力へ渡す仕組み
- `"|"`で表す
#### 使用例：
```js
// echo の出力 → Node プログラムの標準入力へ
echo "hello" | node echo.js
```

## コマンドの実行結果
### `node cat.mjs`
- 予想：ターミナルの入力がそのままターミナルに出力される
- 結果：予想通り
- 何も指定していないため標準入力と標準出力のデフォルトのターミナルが利用された

### `echo FOO | node cat.mjs`
- 予想：FOOがターミナルに出力される
- 結果：予想通り
- パイプラインが使用され `echo` の出力が `node` の標準入力として渡された

### `node cat.mjs > output.txt`
- 予想：ターミナルの入力がoutput.txtに出力される
- 結果：予想通り
- output.txtが標準出力としてリダイレクトされるため

### `node cat.mjs hello.txt`
- 予想：hello.txtの中身がそのままターミナルに出力される
- 結果：予想通り
- hello.txtを標準入力とするため

### `node cat.mjs hello.txt > output.txt`
- 予想：hello.txtの中身がoutput.txtに出力される
- 結果：予想通り
- hello.txtを標準入力として、output.txtにリダイレクトされるため

### `node cat.mjs invalid-file > output.txt`
- 予想：エラーが発生して、output.txtには出力されない
- 結果：予想通り
- 存在しないファイルを標準入力として指定したが、ファイルが見つからないためエラーになる

### `node cat.mjs invalid-file 2> error.txt`
- 予想：error.txtにエラーメッセージが出力される
- 結果：予想通り
- 発生したエラーメッセージが `2>` でerror.txtにリダイレクトされるため
