## 1. index.js でdocument.cookie プロパティを console.logで表示する
### Cookie の中身が表示されない  
**理由**: `sid` Cookie には HttpOnly 属性が設定されており、 JavaScript から読み取りできないようになっているため  
（XSS 攻撃からセッション ID を保護するため）

## 2. ブラウザの開発者コンソールで http://localhost:3000/ の Cookie を表示する
### `sid=20eb64c5-ed2d-4576-8f26-b0e0a2d8ddd2` が表示される  
**理由**: 開発者ツールの Application タブはブラウザの機能であり、HttpOnly Cookie も含めすべての Cookie を確認できるため

## 3. ToDo アプリのタブをリロードする
### リストの状態が保持される  
**理由**: セッション Cookie はブラウザを閉じるまで保持され、サーバー側でセッション ID に紐づいた ToDo リストのデータが保存されているため

## 4. 同一ブラウザの異なるタブやウィンドウで http://localhost:3000/ を開いて ToDo リストの状態を確認する
### リストの状態が共有される  
**理由**: 同じブラウザの同じオリジン（`http://localhost:3000`）では Cookie が共有されるため

## 5. シークレットウィンドウや異なるブラウザで http://localhost:3000/ を開いて ToDo リストの状態を確認する
### リストの状態が共有されない  
**理由**: シークレットウィンドウや異なるブラウザはそれぞれ別の Cookie ストレージを持つため

## 6. http://127.0.0.1:3000/ を開いて ToDo リストの状態を確認する
### リストの状態が共有されない  
**理由**: `localhost` と `127.0.0.1` はホスト名が異なるため別オリジンと見なされ、Cookie が共有されないため
